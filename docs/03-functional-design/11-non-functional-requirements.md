## 1. Executive Summary

This document defines every major interaction a user performs within AegisIQ. It ensures the platform delivers a consistent, intuitive, and explainable experience across all user types.

---

## 2. Journey Overview

## 4. Candidate Journey

### Stage 1 — Registration

```
Landing Page
     ↓
Create Account
     ↓
Complete Profile
     ↓
Select Target Role
```

### Stage 2 — Assessment Preparation

```
Upload Job Description
     ↓
JD Intelligence
     ↓
Role Blueprint
     ↓
Assessment Preview
     ↓
Start Assessment
```

Before the first mission, show the candidate:

- Extracted role
- Required competencies
- Estimated duration
- Mission overview
- Evaluation criteria
- Tips for the assessment

This reduces uncertainty, improves trust, and makes the experience feel more professional.

### Stage 3 — Assessment

```
Mission 1
     ↓
Mission 2
     ↓
Mission 3
     ↓
Mission 4
```

Each mission:

```
Scenario
     ↓
Response
     ↓
Follow-up
     ↓
Completion
```

### Stage 4 — Evaluation

```
Cyber Reasoning
     ↓
Evidence
     ↓
Explainability
     ↓
Competency Profile
```

### Stage 5 — Learning

```
Weak Skills
     ↓
Recommended Topics
     ↓
Labs
     ↓
Learning Plan
     ↓
Next Assessment
```

## 13. Journey Metrics

| Metric | Description |
|---|---|
| Assessment completion rate | Percentage of started assessments that are submitted |
| Mission completion rate | Average missions completed per assessment |
| Average assessment duration | Mean time from start to submission |
| Drop-off rate | Percentage of users who leave at each stage |
| Report downloads | Number of times reports are exported |
| Reassessment rate | Percentage of candidates who take a second assessment |
| Learning roadmap usage | Percentage of candidates who view their learning plan |

## 15. Conclusion

Every journey within AegisIQ should minimize friction while maximizing transparency, learning, and confidence. Users should always understand where they are, what happens next, and why the platform reached its conclusions.

### UX Principles

Every journey should satisfy:

```
Simple

↓

Predictable

↓

Transparent

↓

Adaptive

↓

Explainable

↓

Actionable
```
